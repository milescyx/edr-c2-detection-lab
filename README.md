# EDR Detection Engineering Lab: C2 Access, Credential Dumping, and Automated Response with LimaCharlie

## Executive Summary

This project demonstrates a complete attack-and-detect workflow within a controlled lab environment. A Windows endpoint was intentionally compromised using a Sliver Command and Control (C2) payload hosted from a Kali Linux attack platform. After establishing remote access, post-exploitation activity was conducted to target the Local Security Authority Subsystem Service (LSASS) process and simulate credential theft.

Endpoint telemetry generated by LimaCharlie was used to investigate the attack, identify credential dumping behavior, engineer a custom detection rule, validate the detection against historical telemetry, and ultimately evolve the detection into an automated response capable of terminating malicious processes in real time.

The project demonstrates the practical application of detection engineering principles, incident investigation techniques, and automated endpoint response capabilities using an enterprise-grade EDR platform.

## Project Objectives

The primary objectives of this lab were:

* Simulate endpoint compromise using a Command and Control framework
* Establish remote access to a Windows endpoint
* Execute a credential dumping attack against LSASS
* Investigate malicious behavior using EDR telemetry
* Engineer custom detections within LimaCharlie
* Validate detection logic against attack telemetry
* Implement automated response actions
* Prevent future execution of the attack through process termination

## Attack Scenario

A user downloads and executes a malicious payload disguised as a legitimate executable. The payload establishes a remote session with an attacker-controlled Sliver C2 server.

After validating access, the attacker enumerates running processes and identifies LSASS as a target for credential extraction. A Living-off-the-Land Binary (LOLBIN) technique leveraging rundll32.exe and comsvcs.dll is then used to dump LSASS memory.

The activity is detected through endpoint telemetry generated by LimaCharlie, enabling investigation and response.

# Detection Engineering

## Detection Strategy

The goal was to identify unauthorized access to LSASS while minimizing false positives.

Detection logic focused on:

* Sensitive process access events
* Access to LSASS
* Suspicious parent-child relationships

# Detection Opportunities

Organizations can detect this activity by monitoring:

* SENSITIVE_PROCESS_ACCESS events
* Access to LSASS memory
* Rundll32 execution of comsvcs.dll
* Creation of .dmp files
* Unsigned parent processes spawning LOLBINs

# Lessons Learned

* Credential dumping generates multiple telemetry artifacts beyond process creation events.
* LSASS access is a high-value detection point for EDR platforms.
* Detection validation is critical before production deployment.
* Automated response significantly reduces attacker dwell time.
* LOLBIN-based attacks remain effective and should be continuously monitored.

# Conclusion

This lab demonstrated a full attack lifecycle beginning with endpoint compromise and culminating in credential dumping activity against LSASS. Using LimaCharlie, malicious behavior was identified, investigated, and ultimately prevented through custom detection engineering and automated response actions.

The project highlights how modern EDR platforms can be leveraged not only to detect post-exploitation activity but also to actively disrupt attacker objectives in real time.
